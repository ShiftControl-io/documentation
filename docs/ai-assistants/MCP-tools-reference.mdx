<p style={{fontSize: "large"}}>Every tool the ShiftControl MCP server exposes — read and write.</p>

The ShiftControl MCP server at `https://mcp.shiftcontrol.io/mcp` exposes 11 tools today. Read tools are safe to call any time; write tools require an explicit `confirm: true` argument in the call itself, so destructive operations are always deliberate rather than incidental.

## Read tools

These tools read your ShiftControl data and never modify it. Permissions match your own ShiftControl role — if you can't see a resource in the UI, the AI can't see it either.

| Tool | Purpose | Typical first call |
|------|---------|---------------------|
| `list_my_orgs` | List the ShiftControl organizations your account belongs to. Returns each org's ID and name. | Yes — most other tools need an `orgId`, so call this first. |
| `list_users` | Paginated user list with optional filters (status, role, department, etc.). | "Show me every disabled user." |
| `get_user` | Full detail for a single user by ID — profile, groups, app access, security state. | "What apps does Alex have access to?" |
| `list_groups` | Paginated group list with filter. | "List all groups tagged 'department'." |
| `get_group` | Full detail for one group — members, dynamic rules, app assignments. | "Who's in the Engineering group?" |
| `list_teams` | Paginated team list. | "What teams do we have?" |
| `list_oauth_apps` | Paginated list of OAuth apps your Workspace users have granted access to (the Permissions Dashboard data). | "Which OAuth apps have admin scopes?" |
| `get_oauth_app` | Full detail for one OAuth app — users who granted it, scopes, risk indicators. | "Who granted access to Notion AI?" |
| `list_apps` | Paginated list of managed applications in ShiftControl. | "Show me every SSO app." |
| `get_subscription` | Subscription detail for a managed app — pricing, contract length, renewal. | "What's our Notion subscription look like?" |

## Write tools

Write tools modify ShiftControl data. Each one requires `confirm: true` in the arguments so the AI can't write by accident — a missing `confirm` returns a dry-run preview instead of doing the work.

| Tool | What it changes | Confirmation |
|------|-----------------|--------------|
| `update_subscription` | Updates an app's subscription record — cost, billing cycle, contract length, renewal date. | `confirm: true` required. Calling without `confirm` returns a preview of the diff so you can review before approving. |

## Common usage patterns

**Discover your org**

Ask: "What ShiftControl orgs do I belong to?"

The AI calls `list_my_orgs` first, gets your org ID, then carries that ID through every subsequent call. You don't have to specify it manually.

**List apps and look at one**

Ask: "What apps do we manage in ShiftControl, and what's our Notion record look like?"

The AI calls `list_apps` to enumerate your managed application catalog, finds Notion, then calls `get_subscription` for the subscription detail (cost, billing cycle, renewal). Useful for SaaS spend reviews and renewal prep.

**Find people in a department or team**

Ask: "Who's in the Engineering department?" or "Who's on the Sales team?"

For a department, the AI calls `list_users` with a department filter. For a team, the AI calls `list_teams` to find the team ID, then either `list_users` filtered to that team or `get_group` on the auto-created team group (teams generate matching dynamic groups during onboarding). Same shape works for locations and roles.

**Audit third-party OAuth apps in your Workspace**

Ask: "What third-party apps have access to our Google Workspace, and which have admin scopes?"

The AI calls `list_oauth_apps` to enumerate every OAuth app users have granted access to, then `get_oauth_app` on any one for the full scope list and the users who granted it. This is the OAuth Permissions Dashboard data via MCP.

**Update a subscription from an invoice**

Ask: "I just got the Notion renewal invoice — they bumped us to $1,400/year. Update the subscription record."

If you have an email-search MCP (Gmail, etc.) connected alongside the ShiftControl MCP, the [`refresh-subscription-info` Skill](/ai-assistants/Skills/) automates the whole flow: it pulls the invoice, computes the diff against your current `get_subscription` data, and proposes an `update_subscription` call. Without the Skill, you can still walk through it manually — the AI calls `update_subscription` without `confirm: true` first to show you the diff, then asks for explicit approval. Only after you say "yes" does it call again with `confirm: true`. This is the gating model — destructive actions are always two-step.

## What MCP doesn't cover

The v1 tool surface is intentionally focused. A few common-feeling questions that the current tools can't answer:

- **"Which users haven't logged in for 30 days?"** — Login activity isn't exposed via MCP. Use the Users screen in the product or the Reports section for activity audits.
- **"Which apps haven't been used since March?"** — App usage timestamps aren't on the tool surface yet. The App Spend dashboard in the product has this view.
- **"Which groups have access to AWS?"** — Cross-app access lookups (start from an app, find groups assigned) aren't a direct tool yet. You can get there by listing all groups, then `get_group` to check each one's app assignments — slow for a large org.

These are tracked for future tool additions. Watch [journey.shiftcontrol.io](https://journey.shiftcontrol.io) for new-tool announcements.

## Authentication and scope

Every tool call uses your OAuth-authenticated session. ShiftControl validates the token against PropelAuth on every request. There's no separate API key, no service account, and no way for an AI client to call a tool you don't have permission for — your role in ShiftControl determines what the tool can do.

If your token expires, your AI client refreshes it silently. If refresh fails (you signed out, the org revoked your access, the token was revoked from your account), the next call surfaces a "sign in again" prompt in the AI tool. Re-authorizing is one click in your browser.
